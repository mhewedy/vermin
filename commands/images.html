images

images subcommand work with images which is the blueprint for the VMs.

Once you create a VM from an image for the first time, it is then cached locally for next uses. To list all the cached images, use the command without any parameters: 

$ vermin images
IMAGE NAME                              DISK
generic/centos8                         0.9GB
ubuntu/focal64                          0.5GB

The images are used in the subcommand to create a new VM.

Note: vermin uses the path ~/.vermin/images directory to cache downloaded images.

You can create your own image by submitting an existing VM using the subcommand.

Removing an image

You can remove a downloaded or committed image by using the rmi command.

Note: You need to beware about this feature, as you might remove a local committed image that you cannot recover. 
$ vermin rmi ubuntu/focal64
✔ Deleting image ubuntu/focal64

In case you removed an image that is downloaded from Vagrant Cloud, you can got it downloaded automatically again when you execute the command. e.g.: 

$ vermin create generic/centos8
Image 'generic/centos8' could not be found. Attempting to find and install
✔ virtualbox hypervisor detected
✔ Getting Image info from Vagrant Cloud
Downloading   3% [###...............................................................] (35/926 MB, 5.096 MB/s) [8s:2m46s]
