images

images subcommand work with images which is the blueprint for the VMs.

To list all the available images, use the command without any parameters: 

$ vermin images
IMAGE NAME               CACHED              DISK
ubuntu/focal             true                1.1GB
centos/8
alpine/3.11

the images subcommand list all the images you can use to create a new VM from.

It shows three columns; the image name, cached which means whether this image is saved on the local disk or not. The last column is size on disk of the image.

The images are used in the subcommand to create a new VM.

Note: vermin uses the path ~/.vermin/images directory to cache downloaded images.

You can create your own image by submitting an existing VM using the subcommand.

Purging images cache

Besides the actual image file being downloaded and cached, also the remote Database of the images is downloaded as well.

The images database is downloaded so vermin can work faster. In case of the database changes on the server which is not reflected on the client, the URL of one of the images might be obsolete. In such case you can purge the database cache using the -p|--purge flag. 

$ vermin images -p
IMAGE NAME               CACHED              DISK
ubuntu/focal             true                1.1GB
centos/8
alpine/3.11

This command will purge the database and re-download it (it is usually in Kilobytes) and list the available images in the new database.

Note: Most of the times you will not need the -p flag. As the database is being purged automatically on host machine reboot.

Removing an image

You can remove a downloaded or committed image by using the rmi command.

Note: You need to beware about this feature, as you might remove a local committed image that you cannot recover.
